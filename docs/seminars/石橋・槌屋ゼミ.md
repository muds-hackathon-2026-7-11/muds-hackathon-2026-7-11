## Governance Design とは
- **Governance Design ＝ 政治・経済・社会に対する工学的働きかけ**。
- すなわち、新しい仕組み・枠組み（System）の **設計（Design）・構築（Implementation）・運用（Application）**、そして **実験（Experimentation）と評価（Evaluation）** を伴う一連のプロセス。

## プロジェクトのテーマ：「ITを活用した社会課題解決の検討」
- 2000年代を振り返ると、日常生活の多くの行為を Web に置き換えることで産業構造が大きく変化した。
- Web は依然としてビジネスの場として重要であり、ますますの発展が予想される。
- 一方で、**SDGs に代表される多くの国際的な社会課題は、Web や IT の活用で解決を探れる余地がまだまだある**と考えられる。

## 現在の Web Appli（アプリケーション）の役割
- **Web ＝ ヒトとヒト・モノ・コトを繋ぐ媒体**。起点は「ヒト（＝smartphone）」。
  - ヒト → **ヒト**：SNS、Recruiting など
  - ヒト → **モノ**：E-Commerce、IoT など
  - ヒト → **コト**：Museum、e-Learning、Game、Crowd Funding など
- キーメッセージ：**作り方を学ぶことは勿論大事。でも「ヒトと何をどう繋ぐのか?」というアイディアの設定がさらに大事。**
